# NailSuite — AGENTS.md

## Stack

- **Framework:** TanStack Start (React 19, SSR) via `@tanstack/react-start`
- **Bundler:** Vite 8 + `@vitejs/plugin-react` + `@tailwindcss/vite`
- **Routing:** TanStack Router — **file-based** in `src/routes/`. See `src/routes/README.md` for conventions. `src/routeTree.gen.ts` is auto-generated — do not edit.
- **Styling:** Tailwind CSS v4 (`@import "tailwindcss"` in `styles.css`), no `tailwind.config.*`. All design tokens in `src/styles.css` using `@theme inline` with oklch colors. Dark mode via `.dark` class.
- **State:** TanStack Query (React Query v5) via `QueryClient` in `src/router.tsx`
- **Package manager:** Bun (`bun.lock`, `bunfig.toml`). `bunfig.toml` has 24h supply-chain guard (`minimumReleaseAge = 86400`).
- **Deployment:** Vercel (`vercel.json: framework: tanstack-start`). Nitro bundled with `preset: "vercel"` and `entryFormat: "node"` (workaround for a known SSR issue).
- **TypeScript:** Strict mode, `verbatimModuleSyntax: false`, `noUnusedLocals/Parameters: false`. Path alias `@/*` → `./src/*`.

## Entry points

| File | Role |
|---|---|
| `src/start.ts` | Creates the TanStack Start instance with error middleware |
| `src/server.ts` | Custom fetch handler — imports `@tanstack/react-start/server-entry`, normalizes h3-swallowed SSR errors |
| `src/router.tsx` | Creates TanStack Router + QueryClient |
| `src/routeTree.gen.ts` | Auto-generated by TanStack Router |

bun run dev       # vite dev (SSR dev server)

bun run preview   # vite preview
bun run lint      # eslint .

## Key conventions

### Routes
- API handlers live alongside page routes using TanStack Router's `server.handlers` (e.g., `src/routes/api/public/lead.ts`). POST/OPTIONS handlers defined inline in `createFileRoute`.
- All routes are flat in `src/routes/` — no `pages/` or `app/` directory.
- Single root layout: `src/routes/__root.tsx` with `<Outlet />`.

### Styling
- **Do not** create `tailwind.config.*` or `postcss.config.*` — Tailwind v4 is configured via `@tailwindcss/vite` plugin.
- CSS imports use `?url` suffix: `import appCss from "../styles.css?url"`.
- Custom salon-industry palette: `--ink` (deep), `--rose` (accent), `--blush` (soft), `--gradient-hero`, `--shadow-elegant`, `--shadow-soft` in oklch.

### Components
- shadcn/ui (New York style) in `src/components/ui/`. Icons via lucide-react.
- Utility: `cn()` from `@/lib/utils` (clsx + tailwind-merge).
- Many unused shadcn components exist — avoid adding more and prune when touching.

### Linting & formatting
- ESLint: `eslint.config.js` — typescript-eslint strict, react-hooks, react-refresh (warn on `only-export-components`), Prettier integration.
- **Rule:** `no-restricted-imports` blocks the `server-only` package — use `*.server.ts` naming or `@tanstack/react-start/server-only` instead.
- Prettier: `printWidth: 100`, `semi: true`, `double quotes`, `trailingComma: "all"`.

### Error handling (non-obvious)
- `src/lib/error-capture.ts` binds global `error` and `unhandledrejection` listeners with a 5s TTL to recover original Error objects after h3 swallows them.
- `src/lib/error-page.ts` renders a plain HTML error page (no React).
- `src/server.ts` detects h3's generic "unhandled" JSON response pattern and replaces it with the error page + console log of the captured error.

## Product context

This is a **single-page marketing site** for "NailSuite — The Nail Salon Independence Kit," a white-label nail-salon booking/POS codebase sold to salon owners. Two tiers: DIY License ($797, self-install) and Done-For-You Setup ($5,797, includes deployment).
